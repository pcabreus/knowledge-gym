# Quality bar (minimum acceptable)

A topic is “OK” when it includes:
- Precise definition
- Why it matters (real impact)
- When to use / when not to use (decision rules)
- 2+ real trade-offs
- 1 mini-example (if applicable)
- 3+ trap questions with answers
- Links to prerequisites and related topics

A topic is “Excellent” when it also includes:
- Failure modes (how it breaks in production)
- Concrete anti-patterns
- Signals/observability (metrics/logs)
- “Alternatives and comparison”
